# 分类体系定义

本文件定义舆情数据的分类层级结构，形成一个类似求和树的分类体系。

---

## 分类层级结构

分类体系采用 **5 层结构**：

```
第 1 层：应用（App）
第 2 层：功能模块（Module）
第 3 层：页面/场景（Page）
第 4 层：问题大类（Issue Type）
第 5 层：具体问题（Issue Detail）
```

## 问题类型编码

为便于统计和分析，可为每个分类节点分配编码：

| 层级 | 编码规则 | 示例 |
|------|----------|------|
| 应用 | A + 数字 | A01 = 抖音, A02 = 微信, A03 = 淘宝 |
| 模块 | M + 数字 | M01 = 短视频, M02 = 直播, M03 = 商城 |
| 页面 | P + 数字 | P01 = 视频播放页, P02 = 商品详情页 |
| 问题类型 | T + 数字 | T01 = 性能问题, T02 = 功能问题 |
| 具体问题 | D + 数字 | D01 = 播放卡顿, D02 = 支付失败 |
