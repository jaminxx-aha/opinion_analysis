# 分类推理示例

本文件展示常见问题的分类推理过程，帮助理解如何从用户描述推断具体分类。

---

## 分类格式说明

分类采用三层结构：**`{一级分类}.{二级分类}.{三级分类}`**

- **一级分类**：问题大类（卡顿、响应慢/延迟、闪退/崩溃、启动异常、发热、内存异常、渲染异常、网络异常）
- **二级分类**：问题细分（如卡顿下的滑动卡顿、视频播放卡顿、直播卡顿等）
- **三级分类**：具体场景（如首页推荐视频流上下滑动卡顿）

---

## 示例 1：短视频滑动卡顿

**用户描述：**
> 抖音刷视频卡顿严重

**推理过程：**

1. **提取关键词：** "抖音"、"刷"、"视频"、"卡顿"

2. **确定应用：** 明确提到"抖音" → 应用：抖音

3. **确定一级分类：**
   - "卡顿"是核心问题
   - → 一级分类：卡顿

4. **确定二级分类：**
   - "刷视频"表示滑动浏览操作
   - 滑动操作引发的卡顿属于"滑动卡顿"
   - → 二级分类：滑动卡顿

5. **确定三级分类：**
   - 抖音默认首页是推荐页
   - "刷视频"通常指首页推荐视频流上下滑动
   - → 三级分类：首页推荐视频流上下滑动卡顿

**分类结果：**
```
卡顿.滑动卡顿.首页推荐视频流上下滑动卡顿
```

**置信度：** 高（描述明确，场景清晰）

## 推理技巧总结

### 关键词与分类对应

| 关键词 | 一级分类 | 二级分类方向 |
|--------|---------|-------------|
| 卡顿、卡、顿 | 卡顿 | 根据场景细分 |
| 延迟、慢、滞后 | 响应慢/延迟 | 根据场景细分 |
| 闪退、崩溃、退出 | 闪退/崩溃 | 根据触发时机细分 |
| 打不开、启动慢、加载慢 | 启动异常 | 根据启动类型细分 |
| 发热、烫、温度高 | 发热 | 根据使用场景细分 |
| 内存、占用、泄漏 | 内存异常 | 根据问题类型细分 |
| 模糊、花屏、黑屏、闪烁 | 渲染异常 | 根据对象类型细分 |
| 加载失败、网络、断连 | 网络异常 | 根据功能模块细分 |

### 场景关键词与二级分类

| 场景关键词 | 二级分类（卡顿类） |
|-----------|-------------------|
| 刷、滑动、切换、上下滑 | 滑动卡顿 |
| 看视频、播放、视频画面 | 视频播放卡顿 |
| 进、切到、跳转、进入 | 页面转场卡顿 |
| 直播、看直播 | 直播卡顿 |
| 拍摄、剪辑、编辑、发布 | 拍摄/剪辑卡顿 |
| 点赞、评论、分享、收藏 | 互动操作卡顿 |
| 加载、刷新、首页 | 加载卡顿 |

### 置信度评估

- **高置信度**：描述明确，关键词与场景直接对应
- **中置信度**：描述有歧义，但有合理推断
- **低置信度**：信息不足，需要补充确认
