# 分类推理示例

本文件展示常见问题的分类推理过程，帮助理解如何从用户描述推断具体分类。

---

## 示例 1：短视频卡顿

**用户描述：**
> 手机刷抖音出现卡顿

**推理过程：**

1. **提取关键词：** "手机"、"刷"、"抖音"、"卡顿"

2. **确定应用：** 明确提到"抖音" → 应用：抖音

3. **查阅应用背景：** 抖音包含短视频、直播、商城等模块

4. **推断功能模块：**
   - "刷抖音"是常见口语表达，通常指浏览短视频内容
   - 抖音的核心功能是短视频，"刷"字暗示滑动浏览行为
   - → 模块：短视频

5. **推断具体场景：**
   - 在短视频模块中，"刷"通常指上下滑动切换视频
   - 卡顿可能发生在：
     - 视频播放过程中（播放卡顿）
     - 滑动切换过程中（滑动卡顿）
   - 结合"刷"字，更倾向于滑动操作场景
   - → 页面：视频播放页，场景：滑动操作

6. **确定问题类型：**
   - "卡顿"属于性能问题大类
   - 具体表现为播放或滑动不流畅
   - → 问题类型：性能问题，具体问题：播放卡顿/滑动卡顿

**分类结果：**
```
抖音 > 短视频 > 视频播放页 > 性能问题 > 播放卡顿
```

**置信度：** 中高（"刷抖音卡顿"有多种可能，需要更多信息确定具体场景）

## 推理技巧总结

### 应用识别优先级

分类流程首先检查应用识别状态：

| 识别状态 | 输出 |
|----------|------|
| 无法识别应用名 | `无法识别应用` |
| 应用不在列表中 | `{应用名}没有描述` |
| 应用在列表中 | 继续分类流程 |

### 关键词识别

| 关键词 | 通常对应 |
|--------|----------|
| 刷/看/浏览 | 内容浏览场景 |
| 发/上传/发布 | 内容发布场景 |
| 买/付款/下单 | 购物场景 |
| 卡/慢/顿 | 性能问题 |
| 失败/不行/不到 | 功能问题 |
| 闪退/崩溃/关了 | 崩溃问题 |
| 模糊/不清楚/看不清 | 显示问题 |
| 搜/找/查 | 搜索场景 |
| 聊天/消息/私信 | 聊天场景 |
| 直播/看直播 | 直播场景 |

### 常见推断规则

1. **"刷[APP]"** → 通常指浏览短视频内容
2. **"看直播"** → 直播观看场景
3. **"买东西"** → 购物场景
4. **"卡顿"** → 可能是播放卡顿、滑动卡顿、加载卡顿，需结合场景判断
5. **"加载不出来"** → 内容加载问题，可能是网络或服务器问题
6. **"闪退"** → 应用崩溃，通常与内存或稳定性相关

### 置信度评估

- **高置信度**：描述明确，关键词与场景直接对应
- **中置信度**：描述有歧义，但有合理推断
- **低置信度**：信息不足，需要补充确认
