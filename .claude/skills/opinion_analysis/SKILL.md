# 舆情分析技能

## 概述

本技能用于对舆情数据进行智能分析。当用户提供 Excel 表格形式的舆情数据时，通过结合应用背景知识和分类示例，批量分析用户问题描述，推断问题发生的具体场景，并输出结构化的分类结果和统计汇总。

## 输入格式

### Excel 表格要求

用户提供 Excel 文件时，表格应包含以下列：

| 列名 | 说明 | 示例 |
|------|------|------|
| **问题描述** | 必需列，用户反馈的问题描述文本 | "手机刷抖音出现卡顿" |
| 其他列 | 可选，如用户ID、时间、渠道等 | - |

**Excel 示例结构：**

| 问题描述 | 用户ID | 时间 | 渠道 |
|----------|--------|------|------|
| 手机刷抖音出现卡顿 | 001 | 2024-01-01 | App Store |
| 微信发消息发不出去 | 002 | 2024-01-02 | 官方客服 |
| 淘宝付款失败 | 003 | 2024-01-03 | 12315 |

## 分析流程

### 使用本地脚本（推荐）

本技能提供了本地脚本，可执行基础分类并自动生成可视化报告。

**注意：脚本只处理已知应用名和常见英文别名（如wechat→微信）。对于口语化表达（如"狗东"）、俚语、新应用名，需要大模型手动推断应用名后再使用脚本分类。**

**脚本目录：**
```
scripts/
├── config.py           # 分类配置（应用列表、别名映射）
├── classify_issue.py   # 单条分类脚本
├── analyze_excel.py    # 批量分析脚本（自动生成可视化报告）
└── generate_report.py  # 可视化报告生成脚本
```

**批量分析 Excel（自动生成可视化报告）：**
```bash
python scripts/analyze_excel.py <Excel文件路径>
```

示例：
```bash
python scripts/analyze_excel.py 舆情数据.xlsx
```

**输出文件：**
| 文件 | 说明 |
|------|------|
| `*_result.json` | JSON 格式的分类数据和统计汇总 |
| `*_report.html` | 可视化 HTML 报告（包含图表和详情表格） |

**可视化报告内容：**
- 统计卡片：总条数、成功分类、应用无描述、无法识别
- 应用问题分布图（柱状图）
- 问题类型分布图（饼图）
- 功能模块分布图（横向柱状图）
- 分类状态分布图（饼图）
- 详细分类结果表格（分页显示，每页15条）

**单条分类：**
```bash
python scripts/classify_issue.py <应用名> <问题描述>
```

示例：
```bash
python scripts/classify_issue.py 抖音 "刷抖音视频卡顿严重"
python scripts/classify_issue.py 京东 "搜索功能太卡了"
```

**处理复杂推断的流程：**
1. 先读取 Excel，获取问题描述
2. 大模型逐条推断应用名（处理"狗东"、"拼夕夕"等表达）
3. 更新 Excel 或直接使用推断结果调用脚本分类

### 手动分析流程（备选）

如果需要更精细的分析，可按以下步骤手动处理：

### 步骤 0：读取 Excel 文件

使用 Python pandas 库读取 Excel 文件：

```python
import pandas as pd

# 读取 Excel
df = pd.read_excel("舆情数据.xlsx")

# 获取问题描述列
issues = df["问题描述"].tolist()
```

### 步骤 1：推断应用名

应用推断采用**三层递进策略**：别名映射 → 目录查找 → 模型推理 → 无法识别

---

**第一层：别名映射**

从表格获取应用名，通过常见别名映射转换：

| 原名/别名 | 映射结果 |
|-----------|----------|
| wechat | 微信 |
| douyin | 抖音 |
| taobao | 淘宝 |
| kuaishou | 快手 |
| xiaohongshu / red | 小红书 |
| bilibili / B站 | 哔哩哔哩 |
| jd | 京东 |
| meituan | 美团 |
| pinduoduo / 拼夕夕 | 拼多多 |

---

**第二层：目录查找**

映射后，在 `apps/` 目录查找是否存在对应的应用描述文件：

```
apps/
├── 抖音.md
├── 微信.md
├── 淘宝.md
├── 快手.md
└── 小红书.md
```

- **找到** → 使用该应用名，继续分类流程
- **未找到** → 进入第三层模型推理

---

**第三层：模型推理**

对于映射后仍找不到的应用名，由大模型进行智能联想推断：

**品牌别名/俚语推断：**
| 用户表达 | 推断应用 |
|----------|----------|
| 狗东、东哥 | 京东 |
| 阿里爸爸 | 淘宝 |
| 腾讯、鹅厂 | 微信 |
| 字节、头条系 | 抖音 |
| 小破站 | 哔哩哔哩 |
| 小黄 | 美团 |

**功能特征推断：**
| 关键词 | 推断应用 |
|--------|----------|
| 刷短视频 | 抖音/快手 |
| 朋友圈 | 微信 |
| 笔记、种草 | 小红书 |
| 外卖、配送 | 美团 |
| 弹幕、番剧 | 哔哩哔哩 |

---

**第四层：无法识别**

如果模型推理后：
- **不确定**：推断结果置信度低，或多个候选无法确定
- **不知道**：完全不认识该表达或应用

则输出：`无法识别的应用`

---

**推断流程图：**

```
表格应用名
    ↓
别名映射 (wechat → 微信, 狗东 → 狗东)
    ↓
查找 apps/ 目录
    ↓
┌───────────────────────────────────────────┐
│ 找到 → 继续分类流程                         │
│                                            │
│ 未找到 → 大模型推理                         │
│     ↓                                      │
│ 推理确定 → 输出"{推断应用名}没有描述"        │
│ 推理不确定/不知道 → 输出"无法识别的应用"     │
└───────────────────────────────────────────┘
```

---

**示例推断过程：**

| 输入应用名 | 映射结果 | 目录查找 | 模型推理 | 最终输出 |
|------------|----------|----------|----------|----------|
| wechat | 微信 | ✅ 找到 | - | 继续分类 |
| 抖音 | 抖音 | ✅ 找到 | - | 继续分类 |
| 狗东 | 狗东(无映射) | ❌ 未找到 | → 京东 | 京东没有描述 |
| 拼夕夕 | 拼多多 | ❌ 未找到 | - | 拼多多没有描述 |
| 某某APP | 无映射 | ❌ 未找到 | 不确定 | 无法识别的应用 |
| abcd123 | 无映射 | ❌ 未找到 | 不知道 | 无法识别的应用 |

---

### 步骤 2：分析问题描述

结合应用描述，分析用户的问题：
- 功能模块（如短视频、直播、商城等）
- 问题类型（如卡顿、崩溃、加载失败等）
- 发生场景（如播放、滑动、搜索等）

### 步骤 3：参考分类示例

查阅 [examples.md](examples.md) 了解常见问题的分类推理过程。

### 步骤 4：确定分类路径

查阅 [taxonomy.md](taxonomy.md) 找到最匹配的分类层级路径。

### 步骤 5：输出结果

对每条数据进行分类，并生成汇总统计。
