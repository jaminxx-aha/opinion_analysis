## 概述

本技能用于对舆情数据进行智能分析。当用户提供 Excel 表格形式的舆情数据时，通过结合应用背景知识和分类示例，批量分析用户问题描述，推断问题发生的具体场景，并输出结构化的分类结果和统计汇总。

## 分类格式

**采用三层分类结构：**

```
{一级分类}.{二级分类}.{三级分类}
```

**示例：**
- `卡顿.滑动卡顿.首页推荐视频流上下滑动卡顿`
- `响应慢/延迟.直播延迟.直播画面延迟`
- `闪退/崩溃.使用过程闪退.视频播放过程闪退`

**分类层级说明：**

| 层级 | 说明 | 示例 |
|------|------|------|
| 一级分类 | 问题大类 | 卡顿、响应慢/延迟、闪退/崩溃、启动异常、发热、内存异常、渲染异常、网络异常 |
| 二级分类 | 问题细分 | 滑动卡顿、视频播放卡顿、直播卡顿、页面转场卡顿等 |
| 三级分类 | 具体场景 | 首页推荐视频流上下滑动卡顿、直播观看画面卡顿等 |

## 输入格式

### Excel 表格要求

用户提供 Excel 文件时，**需要先识别列名**：

**识别步骤：**

1. **调用脚本获取Excel信息**：
   ```bash
   python scripts/analyze_excel.py <Excel文件路径> --info
   ```

2. **根据返回的列名和样本数据，判断哪个列是应用名列、哪个是问题描述列**：

   **应用名列识别依据：**
   - 列内容包含已知应用名（如"抖音"、"微信"、"淘宝"、"wechat"、"douyin"等）
   - 列内容较短，通常只有应用名称
   - 列名可能包含"应用"、"app"、"平台"、"软件"等关键词

   **问题描述列识别依据：**
   - 列内容包含问题描述文本（如"抖音刷视频卡顿"、"微信发不出消息"）
   - 列内容较长，包含具体问题描述
   - 列内容包含问题关键词（如"卡顿"、"慢"、"失败"、"加载"、"打不开"等）
   - 列名可能包含"问题"、"描述"、"反馈"、"投诉"、"内容"等关键词

   **如果无法识别：**
   - 如果找不到包含应用名的列，应用名列可以不指定（后续从问题描述中推断）
   - 如果找不到包含问题描述列，返回 **"Excel格式错误：无法识别问题描述列"**

   识别完成后，记录应用名列索引号和问题描述列索引号（从1开始），后续步骤使用 `fetch_data.py` 通过这两个索引号分批读取数据。

**Excel 示例结构：**

| 应用名 | 问题描述 | 用户ID | 时间 | 渠道 |
|--------|----------|--------|------|------|
| 抖音 | 手机刷抖音出现卡顿 | 001 | 2024-01-01 | App Store |
| wechat | 微信发消息发不出去 | 002 | 2024-01-02 | 官方客服 |
| 淘宝 | 淘宝付款失败 | 003 | 2024-01-03 | 12315 |

**已知应用名和别名：** 完整别名映射见 [scripts/config.py](scripts/config.py) 的 `app_alias_map`。

**说明：** apps/ 目录下仅包含抖音、微信、淘宝、快手、小红书的描述文件。哔哩哔哩、京东、美团、拼多多等应用虽有别名映射但缺少描述文件，分类时会返回 `no_description` 状态。

## 分析流程

### 核心分类步骤

**分类判断由子Agent分批完成：**

每批5行数据，子Agent通过脚本读取数据、分类体系和应用描述，避免主会话上下文膨胀。

**子Agent任务流程：**

1. 调用 `fetch_data.py --json` 读取当前批次的数据（包含所有列的原始数据）
2. 读取 `taxonomy.md` 获取完整分类树和歧义说明
3. 读取 `apps/<应用名>.md` 获取应用描述
4. 对每条数据分类，合并raw_data输出固定格式JSON
5. 调用 `save_classification.py` 将分类结果和原始数据追加到 prepared.db

**子Agent分类提示词格式：**

```
请对以下舆情数据进行分类：

## 读取数据
调用脚本读取当前批次数据（JSON格式，包含所有列原始数据）：
python scripts/fetch_data.py <Excel文件路径> --app-column <app列索引> --problem-column <problem列索引> --start <起始行> --end <结束行> --json

## 分类体系
读取 taxonomy.md 获取完整分类体系（8个一级分类的全部二级、三级分类）和歧义说明

## 应用描述
根据数据中的应用名，读取 apps/<应用名>.md 获取应用描述

分类格式：{一级分类}.{二级分类}.{三级分类}

请返回分类结果，格式为JSON数组（保留fetch_data.py返回的raw_data字段）：
[
  {
    "row_index": 行号,
    "app": "应用名",
    "problem": "问题描述",
    "raw_data": { ... },  // 保留fetch_data.py返回的原始数据，不要修改
    "classification": {
      "app": "应用名",
      "level1": "一级分类",
      "level2": "二级分类",
      "level3": "三级分类",
      "full_path": "一级分类.二级分类.三级分类"
    }
  }
]

## 保存分类结果
分类完成后，调用脚本将结果保存到数据库：
python scripts/save_classification.py '<JSON结果>' --output-dir <output_dir> --excel-source <Excel文件路径>

注意：--excel-source 仅在首次调用时需要提供（用于创建数据库并读取原始数据行），后续批次无需提供。
```

## 输出路径处理

**在开始分析前，需要确定输出目录：**

```
步骤0: 确定输出路径
┌─────────────────────────────────────────────────────────────┐
│ 输入: 用户可能指定输出路径，也可能不指定                        │
│                                                             │
│ 处理逻辑:                                                    │
│ 1. 如果用户指定了输出路径:                                    │
│    - 检查该路径是否存在                                       │
│    - 如果存在且是文件夹 → output_path = 用户指定路径           │
│    - 如果存在但不是文件夹 → 报错终止: "输出路径不是文件夹"       │
│    - 如果不存在 → 创建该文件夹，output_path = 用户指定路径      │
│                                                             │
│ 2. 如果用户未指定输出路径:                                    │
│    - output_path = ./output                                  │
│    - 如果该路径不存在 → 创建 ./output 文件夹                   │
│                                                             │
│ 3. 在 output_path 下创建以Excel文件名命名的子目录:             │
│    - excel_basename = Excel文件名（去除.xlsx/.xls后缀）        │
│    - output_dir = output_path/excel_basename                  │
│    - 创建 output_dir 目录                                     │
│                                                             │
│ 4. 复制原始Excel文件到output_dir:                             │
│    - 将原始Excel文件复制一份到output_dir中                     │
│    - 用于HTML报告中提供下载链接                               │
│                                                             │
│ 5. 将 output_dir 传递给后续脚本调用                           │
│                                                             │
│ 输出目录结构:                                                 │
│   output_dir/                                                │
│   ├── 原始Excel文件.xlsx        (复制，用于下载)              │
│   ├── xxx_prepared.db           (分类累积数据库)              │
│   └── xxx_report.html            (可视化报告)                 │
└─────────────────────────────────────────────────────────────┘
```

**示例：**

| 场景 | Excel文件 | 用户指定路径 | 最终output_dir |
|------|-----------|-------------|----------------|
| 用户指定路径且存在 | data.xlsx | /home/reports | /home/reports/data |
| 用户指定路径不存在 | data.xlsx | /home/new | 创建后: /home/new/data |
| 用户未指定 | data.xlsx | 无 | ./output/data |
| 用户指定但非文件夹 | data.xlsx | /home/existing.txt | **报错终止** |

### 使用脚本辅助

本技能提供了辅助脚本用于读取Excel、生成可视化报告等。

**脚本目录：**
```
scripts/
├── config.py           # 分类配置（应用列表、别名映射、分类规则）
├── analyze_excel.py    # Excel信息查看和报告生成脚本
├── fetch_data.py       # 分批提取Excel应用名和问题描述列数据
├── save_classification.py # 追加分类结果到prepared.db
├── generate_report.py  # 可视化报告生成脚本
└── read_excel.py       # Excel表格数据通用读取脚本
```

**脚本参数说明：**

| 脚本 | 参数 | 说明 |
|------|------|------|
| analyze_excel.py | `--info` | 显示Excel列名（带索引号）和前10行样本数据 |
| fetch_data.py | `--app-column` | 应用名列索引（从1开始）或列名 |
| fetch_data.py | `--problem-column` | 问题描述列索引（从1开始）或列名 |
| fetch_data.py | `--start / --end` | 数据行范围（从1开始，不含表头） |
| save_classification.py | `<json_input>` | 分类结果JSON字符串或文件路径 |
| save_classification.py | `--output-dir` | 输出目录（prepared.db所在目录） |

**批量分析流程：**

```
步骤0: 确定输出路径（见上方"输出路径处理"）
┌─────────────────────────────────────────────────────────────┐
│ 确定 output_dir，用于存放所有生成的文件                        │
└─────────────────────────────────────────────────────────────┘

步骤1: 获取Excel信息，判断列名
┌─────────────────────────────────────────────────────────────┐
│ python scripts/analyze_excel.py <Excel文件路径> --info      │
│                                                             │
│ 返回: 列名列表 + 各列前10行样本数据 + 总行数                  │
│                                                             │
│ 根据样本数据判断:                                            │
│ - 应用名列索引号（如 [2]）                                   │
│ - 问题描述列索引号（如 [5]）                                  │
│                                                             │
│ 如果无法识别问题描述列，返回: Excel格式错误                   │
└─────────────────────────────────────────────────────────────┘

步骤2: 分批读取数据并分类（每批5行）
┌─────────────────────────────────────────────────────────────┐
│ 计算批次: 从 --info 获取总行数，每批5行                      │
│                                                             │
│ 对每个批次（如行1-5, 行6-10, ...）:                          │
│                                                             │
│ 1. 创建子Agent，指示其:                                      │
│    a. 调用 fetch_data.py 读取该批次数据                      │
│    b. 读取 taxonomy.md 获取分类体系                           │
│    c. 读取 apps/<应用名>.md 获取应用描述                      │
│    d. 对每行数据分类，输出固定格式JSON数组                     │
│    e. 调用 save_classification.py 保存结果到数据库            │
│                                                             │
│    首次批次需传 --excel-source 参数:                          │
│    python scripts/save_classification.py '<JSON>'            │
│      --output-dir <output_dir>                               │
│      --excel-source <Excel文件路径>                           │
│                                                             │
│    后续批次无需 --excel-source:                               │
│    python scripts/save_classification.py '<JSON>'            │
│      --output-dir <output_dir>                               │
│                                                             │
│ 2. 子Agent返回保存确认结果                                   │
│                                                             │
│ 重复直至所有批次完成                                         │
└─────────────────────────────────────────────────────────────┘

步骤3: 生成可视化报告
┌─────────────────────────────────────────────────────────────┐
│ python scripts/analyze_excel.py <prepared.db路径> \         │
│   --output-dir <output_dir>                                 │
│                                                             │
│ 输出: <output_dir>/xxx_report.html                          │
└─────────────────────────────────────────────────────────────┘
```

## 使用示例

**用户输入：** "请分析这个舆情数据表格：舆情数据.xlsx"

**处理流程：** 读取Excel信息 → 识别列名 → 分批分类 → 生成报告

**示例分类结果：**
- "手机刷抖音出现卡顿" → `卡顿.滑动卡顿.首页推荐视频流上下滑动卡顿`
- "抖音直播画面模糊" → `渲染异常.直播渲染异常.直播画面模糊`
- "微信发消息发不出去" → `网络异常.交互网络异常.消息发送失败`
- "手机很卡用不了" → 无法识别应用
- "微博发不了内容" → 微博没有描述

完整输出格式和 JSON 结构见 [taxonomy.md](taxonomy.md)。
