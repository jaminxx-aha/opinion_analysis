## 概述

本技能用于对舆情数据进行智能分析。当用户提供 Excel 表格形式的舆情数据时，通过结合应用背景知识和分类示例，批量分析用户问题描述，推断问题发生的具体场景，并输出结构化的分类结果和统计汇总。

## 分类格式

**采用三层分类结构：**

```
{一级分类}.{二级分类}.{三级分类}
```

**示例：**
- `卡顿.滑动卡顿.首页推荐视频流上下滑动卡顿`
- `响应慢/延迟.直播延迟.直播画面延迟`
- `闪退/崩溃.使用过程闪退.视频播放过程闪退`

**分类层级说明：**

| 层级 | 说明 | 示例 |
|------|------|------|
| 一级分类 | 问题大类 | 卡顿、响应慢/延迟、闪退/崩溃、启动异常、发热、内存异常、渲染异常、网络异常 |
| 二级分类 | 问题细分 | 滑动卡顿、视频播放卡顿、直播卡顿、页面转场卡顿等 |
| 三级分类 | 具体场景 | 首页推荐视频流上下滑动卡顿、直播观看画面卡顿等 |

## 输入格式

### Excel 表格要求

用户提供 Excel 文件时，**需要先识别列名**：

**识别步骤：**

1. **调用脚本获取Excel信息**：
   ```bash
   python scripts/analyze_excel.py <Excel文件路径> --info
   ```

2. **根据返回的列名和样本数据，判断哪个列是应用名列、哪个是问题描述列**：

   **应用名列识别依据：**
   - 列内容包含已知应用名（如"抖音"、"微信"、"淘宝"、"wechat"、"douyin"等）
   - 列内容较短，通常只有应用名称
   - 列名可能包含"应用"、"app"、"平台"、"软件"等关键词

   **问题描述列识别依据：**
   - 列内容包含问题描述文本（如"抖音刷视频卡顿"、"微信发不出消息"）
   - 列内容较长，包含具体问题描述
   - 列内容包含问题关键词（如"卡顿"、"慢"、"失败"、"加载"、"打不开"等）
   - 列名可能包含"问题"、"描述"、"反馈"、"投诉"、"内容"等关键词

   **如果无法识别：**
   - 如果找不到包含应用名的列，应用名列可以不指定（后续从问题描述中推断）
   - 如果找不到包含问题描述的列，返回 **"Excel格式错误：无法识别问题描述列"**

3. **使用识别出的列名准备数据**（支持列索引号，避免中文编码问题）：
   ```bash
   # 使用列索引号（推荐，从1开始）
   python scripts/analyze_excel.py <Excel文件路径> --prepare-only \
     --app-column 1 \
     --problem-column 2

   # 或使用列名（可能有编码问题）
   python scripts/analyze_excel.py <Excel文件路径> --prepare-only \
     --app-column "<应用名列名>" \
     --problem-column "<问题描述列名>"
   ```

   **列索引号说明：**
   - `--info` 输出会显示每个列的索引号，如 `[1] 应用名`、`[2] 问题描述`
   - 使用数字索引可避免Windows环境下中文参数编码问题
   - 索引从1开始

**Excel 示例结构：**

| 应用名 | 问题描述 | 用户ID | 时间 | 渠道 |
|--------|----------|--------|------|------|
| 抖音 | 手机刷抖音出现卡顿 | 001 | 2024-01-01 | App Store |
| wechat | 微信发消息发不出去 | 002 | 2024-01-02 | 官方客服 |
| 淘宝 | 淘宝付款失败 | 003 | 2024-01-03 | 12315 |

**已知应用名和别名：**

| 抖音 | douyin、字节、头条 |
| 微信 | wechat、腾讯 |
| 淘宝 | taobao、阿里 |
| 小红书 | xiaohongshu、red |

## 分析流程

### 核心分类步骤

**分类判断由子Agent完成：**

对于每条舆情数据，分类步骤如下：

1. **推断应用名**：从问题描述或表格中推断应用名称
2. **读取应用描述**：如果应用在 `apps/` 目录中有描述文件，读取该文件内容
3. **读取分类体系**：从 `taxonomy.md` 和 `format.md` 中提取分类体系
4. **创建子Agent判断分类**：将"问题描述"、"应用描述"、"分类体系"发送给子Agent
5. **返回分类结果**：子Agent返回三层分类路径

**子Agent分类提示词格式：**

```
请根据以下信息对用户反馈的问题进行分类：

## 应用信息
应用名称：{app_name}
应用描述：
{app_description}

## 分类体系（三层结构）
格式：{一级分类}.{二级分类}.{三级分类}

一级分类：卡顿、响应慢/延迟、闪退/崩溃、启动异常、发热、内存异常、渲染异常、网络异常

二级分类（卡顿类）：
- 滑动卡顿：首页推荐视频流上下滑动卡顿、关注页视频流滑动卡顿、同城页视频流滑动卡顿
- 视频播放卡顿：首页推荐视频播放卡顿、关注页视频播放卡顿、搜索结果视频播放卡顿
- 直播卡顿：直播观看画面卡顿、直播弹幕滚动卡顿、直播礼物动画卡顿
- 页面转场卡顿：首页进入直播间转场卡顿、视频页进入评论区转场卡顿
- 拍摄/剪辑卡顿：视频拍摄预览卡顿、视频剪辑编辑卡顿
- 互动操作卡顿：视频点赞动画卡顿、视频评论发送卡顿
- 加载卡顿：首页刷新加载卡顿、视频封面加载卡顿

## 用户问题描述
{problem_description}

请返回最合适的分类结果，格式为JSON：
{
  "level1": "一级分类",
  "level2": "二级分类",
  "level3": "三级分类",
  "full_path": "一级分类.二级分类.三级分类",
  "reasoning": "分类理由"
}
```

## 输出路径处理

**在开始分析前，需要确定输出目录：**

```
步骤0: 确定输出路径
┌─────────────────────────────────────────────────────────────┐
│ 输入: 用户可能指定输出路径，也可能不指定                        │
│                                                             │
│ 处理逻辑:                                                    │
│ 1. 如果用户指定了输出路径:                                    │
│    - 检查该路径是否存在                                       │
│    - 如果存在且是文件夹 → output_path = 用户指定路径           │
│    - 如果存在但不是文件夹 → 报错终止: "输出路径不是文件夹"       │
│    - 如果不存在 → 创建该文件夹，output_path = 用户指定路径      │
│                                                             │
│ 2. 如果用户未指定输出路径:                                    │
│    - output_path = ./output                                  │
│    - 如果该路径不存在 → 创建 ./output 文件夹                   │
│                                                             │
│ 3. 在 output_path 下创建以Excel文件名命名的子目录:             │
│    - excel_basename = Excel文件名（去除.xlsx/.xls后缀）        │
│    - output_dir = output_path/excel_basename                  │
│    - 创建 output_dir 目录                                     │
│                                                             │
│ 4. 复制原始Excel文件到output_dir:                             │
│    - 将原始Excel文件复制一份到output_dir中                     │
│    - 用于HTML报告中提供下载链接                               │
│                                                             │
│ 5. 将 output_dir 传递给后续脚本调用                           │
│                                                             │
│ 输出目录结构:                                                 │
│   output_dir/                                                │
│   ├── 原始Excel文件.xlsx        (复制，用于下载)              │
│   ├── xxx_prepared.json         (准备数据)                   │
│   ├── xxx_classified.json        (分类结果)                   │
│   └── xxx_report.html            (可视化报告)                 │
└─────────────────────────────────────────────────────────────┘
```

**示例：**

| 场景 | Excel文件 | 用户指定路径 | 最终output_dir |
|------|-----------|-------------|----------------|
| 用户指定路径且存在 | data.xlsx | /home/reports | /home/reports/data |
| 用户指定路径不存在 | data.xlsx | /home/new | 创建后: /home/new/data |
| 用户未指定 | data.xlsx | 无 | ./output/data |
| 用户指定但非文件夹 | data.xlsx | /home/existing.txt | **报错终止** |

### 使用脚本辅助

本技能提供了辅助脚本用于读取Excel、生成可视化报告等。

**脚本目录：**
```
scripts/
├── config.py           # 分类配置（应用列表、别名映射、分类规则）
├── analyze_excel.py    # 数据准备和报告生成脚本
└── generate_report.py  # 可视化报告生成脚本
```

**脚本参数说明：**

| 参数 | 说明 |
|------|------|
| `--info` | 显示Excel列名（带索引号）和前10行样本数据 |
| `--prepare-only` | 准备数据供分类使用（需指定列） |
| `--app-column` | 应用名列索引或列名（如 `1` 或 `应用名`） |
| `--problem-column` | 问题描述列索引或列名（必需，如 `2` 或 `问题描述`） |
| `--output-dir` | 指定输出目录（**使用步骤0确定的output_dir**） |

**批量分析流程：**

```
步骤0: 确定输出路径（见上方"输出路径处理"）
┌─────────────────────────────────────────────────────────────┐
│ 确定 output_dir，用于存放所有生成的文件                        │
└─────────────────────────────────────────────────────────────┘

步骤1: 获取Excel信息，判断列名
┌─────────────────────────────────────────────────────────────┐
│ python scripts/analyze_excel.py <Excel文件路径> --info      │
│                                                             │
│ 返回: 列名列表 + 各列前10行样本数据                           │
│                                                             │
│ 根据样本数据判断:                                            │
│ - 应用名列: 内容包含"抖音"、"微信"等应用名                    │
│ - 问题描述列: 内容包含具体问题描述文本                        │
│                                                             │
│ 如果无法识别问题描述列，返回: Excel格式错误                   │
└─────────────────────────────────────────────────────────────┘

步骤2: 使用识别出的列名准备数据
┌─────────────────────────────────────────────────────────────┐
│ python scripts/analyze_excel.py <Excel文件路径> \           │
│   --prepare-only \                                          │
│   --app-column "<应用名列名>" \                             │
│   --problem-column "<问题描述列名>" \                        │
│   --output-dir <output_dir>                                 │
│                                                             │
│ 输出: <output_dir>/xxx_prepared.json                         │
└─────────────────────────────────────────────────────────────┘

步骤3: 子Agent对每条数据进行分类
┌─────────────────────────────────────────────────────────────┐
│ 1. 读取 <output_dir>/xxx_prepared.json 文件                  │
│                                                             │
│ 2. 对每条数据调用子Agent进行分类（见核心分类步骤）              │
│                                                             │
│ 3. 将分类结果写入每条数据的 classification 字段：            │
│    {                                                        │
│      "row_index": 1,                                        │
│      "app": "抖音",                                          │
│      "problem": "问题描述",                                  │
│      "classification": {  // ← 新的三层分类格式              │
│        "app": "抖音",                                        │
│        "level1": "卡顿",                                     │
│        "level2": "滑动卡顿",                                 │
│        "level3": "首页推荐视频流上下滑动卡顿",                │
│        "full_path": "卡顿.滑动卡顿.首页推荐视频流上下滑动卡顿" │
│      }                                                      │
│    }                                                        │
│                                                             │
│ 4. 保存更新后的 xxx_prepared.json 文件                       │
│                                                             │
│ 注意: 不要直接创建 xxx_classified.json 文件！                 │
│       分类结果应写入 _prepared.json 的 items 数组中           │
│       脚本会自动生成 _classified.json 和 _report.html         │
└─────────────────────────────────────────────────────────────┘

步骤4: 生成可视化报告
┌─────────────────────────────────────────────────────────────┐
│ python scripts/analyze_excel.py <prepared.json路径> \       │
│   --output-dir <output_dir>                                 │
│                                                             │
│ 输出: <output_dir>/xxx_classified.json                       │
│       <output_dir>/xxx_report.html                          │
└─────────────────────────────────────────────────────────────┘
└─────────────────────────────────────────────────────────────┘
```
