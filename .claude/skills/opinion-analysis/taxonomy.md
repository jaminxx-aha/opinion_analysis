# 分类体系定义

分类格式：`{一级分类}.{二级分类}.{三级分类}`，层级递进，逐层细化问题原因。

8个一级分类对所有应用通用：卡顿、响应慢/延迟、闪退/崩溃、启动异常、发热、内存异常、渲染异常、网络异常。

具体的二级、三级分类和推理示例见各应用描述文件 `apps/<应用名>.md`，分类时只加载对应应用的描述文件即可。

## 分类使用说明

### 1. 分类选择原则

- **从大到小**：先确定一级大类，再细分二级、三级
- **场景匹配**：三级分类必须对应具体应用场景
- **优先核心问题**：当问题涉及多个分类时，优先选择核心问题类型

### 2. 扩展分类

当遇到新场景不在当前分类体系中时，可按以下规则扩展：

1. 保持一级、二级分类不变
2. 在三级分类中新增具体场景
3. 新增分类需遵循命名规范：`{页面/功能}.{具体操作}.{问题表现}`

## 单条分类结果格式

**成功分类：**
```json
{
  "input": "问题描述",
  "status": "success",
  "classification": {
    "app": "应用名",
    "level1": "一级分类",
    "level2": "二级分类",
    "level3": "三级分类",
    "full_path": "一级分类.二级分类.三级分类"
  }
}
```

**不属于8类性能问题：**
```json
{
  "input": "问题描述",
  "status": "unrecognized",
  "output": "不属于性能问题分类"
}
```

**应用没有描述文件：**
```json
{
  "input": "问题描述",
  "status": "no_description",
  "output": "{应用名}没有描述"
}
```

## 关键词与分类对应

| 关键词 | 一级分类 | 二级分类方向 |
|--------|---------|-------------|
| 卡顿、卡、顿 | 卡顿 | 根据场景细分 |
| 延迟、慢、滞后 | 响应慢/延迟 | 根据场景细分 |
| 闪退、崩溃、退出 | 闪退/崩溃 | 根据触发时机细分 |
| 打不开、启动慢、加载慢 | 启动异常 | 根据启动类型细分 |
| 发热、烫、温度高 | 发热 | 根据使用场景细分 |
| 内存、占用、泄漏 | 内存异常 | 根据问题类型细分 |
| 模糊、花屏、黑屏、闪烁 | 渲染异常 | 根据对象类型细分 |
| 加载失败、网络、断连 | 网络异常 | 根据功能模块细分 |
